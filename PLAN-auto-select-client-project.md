# Plan: Auto-select Client/Project Based on Daily Work

## Context

The timesheet bot currently uses a single CLIENT_ID/PROJECT_ID for the entire week. The user works across 2 clients:
- **DAT005 - DATACENTRIX (PTY) LTD** (ID=429) — for daas-*, dcx-*, nlu-*, internal work
- **ENET001 - ENETWORKS FTTH** (ID=2859) — for enetworks-*, goturbo-* projects

The user wants the bot to automatically determine which client/project to log per day based on Claude CLI history and calendar events, and split hours equally across all distinct projects worked on that day (8h / N projects).

---

## Implementation

### 1. Add `client_catalog.json` — cached S-Cubed hierarchy

A new `discover_all` command walks ALL clients → projects → activities → designations non-interactively and saves to `client_catalog.json`:

```json
{
  "employee_id": 11266,
  "entity_id": 1,
  "clients": [
    {
      "id": 429, "name": "DAT005 - DATACENTRIX (PTY) LTD",
      "projects": [
        {"id": 20, "name": "Project X", "activity_id": 1, "designation_id": 1}
      ]
    },
    {
      "id": 2859, "name": "ENET001 - ENETWORKS FTTH",
      "projects": [...]
    }
  ]
}
```

### 2. Add `project_mappings.json` — work folder → S-Cubed project mapping

User-configurable file that maps local folder names/patterns to S-Cubed client+project:

```json
{
  "default_client_id": 429,
  "default_project_id": null,
  "mappings": [
    {"pattern": "enetworks*", "client_id": 2859, "project_id": null},
    {"pattern": "goturbo*", "client_id": 2859, "project_id": null},
    {"pattern": "daas*", "client_id": 429, "project_id": null},
    {"pattern": "dcx*", "client_id": 429, "project_id": null},
    {"pattern": "nlu*", "client_id": 429, "project_id": null}
  ]
}
```

`project_id: null` means "use the first project under that client" (auto-resolved from catalog).

### 3. Modify `build_entry()` to accept per-entry IDs

Change signature from using globals to accepting explicit IDs:

```python
def build_entry(date, hours, comment, client_id, project_id, activity_id, designation_id):
```

### 4. Modify `create_week()` — per-day multi-project logic

For each working day:
1. Get Claude projects list for that day
2. Get calendar events for that day
3. Match each project/event to a S-Cubed client via `project_mappings.json`
4. Group by unique (client_id, project_id) combos
5. Split hours equally: `HOURS_PER_DAY / num_distinct_projects`
6. Create one entry per distinct project with its share of hours and relevant comment
7. If no work found, fall back to default client with full 8h

### 5. New command: `discover_all`

- Walks all clients non-interactively (no `input()`)
- For each client, fetches projects, activities, designations
- Auto-selects first activity/designation if multiple exist
- Saves full catalog to `client_catalog.json`
- Generates initial `project_mappings.json` based on folder name heuristics
